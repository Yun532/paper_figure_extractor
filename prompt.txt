You are building a reusable Claude skill called:

paper_figure_extractor

Goal:
Extract REAL figures from scientific papers (arXiv, PDFs, journal pages such as Nature) with strict non-hallucination guarantees.

This must be a robust, reusable skill—not a one-off script.

==================================================
HARD NON-HALLUCINATION RULES (MANDATORY)
==================================================

- NEVER fabricate figures.
- NEVER fabricate captions.
- NEVER fabricate figure numbers.
- NEVER fabricate panel labels (a, b, c).
- NEVER generate or redraw images.
- NEVER infer missing panels without evidence.

If anything is uncertain:
→ mark it as "uncertain"
→ save as candidate output
→ DO NOT promote to final FigXX

Every output MUST have provenance:
- source file, LaTeX, HTML, or PDF location

==================================================
INPUT TYPES
==================================================

The tool must support:

1. arXiv ID or URL
2. Local PDF
3. Journal article URL (especially Nature-style pages)

==================================================
OUTPUT STRUCTURE
==================================================

OUTPUT/
  metadata.json
  figures/
    Fig01.png
    Fig01_panel_a.png (only if verified)
    Fig02.png
  candidates/
    Fig03_candidate_page_5.png
    Fig04_panel_candidate_01.png
  captions/
    Fig01.txt
    Fig02.txt

==================================================
RELIABILITY HIERARCHY (STRICT ORDER)
==================================================

Always prefer higher-confidence sources:

Level 1 (HIGH):
- arXiv source LaTeX
- parse \begin{figure}, \includegraphics, \caption

Level 2:
- HTML <figure> and <figcaption> from journal pages

Level 3:
- PDF embedded image objects

Level 4 (LOW):
- PDF page rendering + caption-based cropping

Lower levels MUST NOT override higher ones.

==================================================
FIGURE EXTRACTION LOGIC
==================================================

1. arXiv source path:
   - download e-print
   - find main .tex
   - parse figure environments
   - extract includegraphics paths
   - extract caption text
   - assign Fig01, Fig02 by order

2. HTML path:
   - parse <figure>
   - extract image URLs
   - extract caption
   - map "Fig. X" to FigXX

3. PDF path:
   - extract embedded images (PyMuPDF or pdfimages)
   - if not sufficient:
       → render pages
       → locate "Fig." or "Figure" text
       → generate candidate crops ONLY

==================================================
PANEL EXTRACTION RULES
==================================================

Panel extraction must follow strict evidence levels:

Level A (HIGH confidence):
- multiple \includegraphics inside one figure environment
- multiple images inside one HTML figure
→ export as:
  Fig01_panel_a.png
  Fig01_panel_b.png

Level B (MEDIUM):
- caption explicitly references (a), (b), (c)
- panel labels detected clearly in image
→ assign panel labels only if alignment is reliable

Level C (LOW / UNCERTAIN):
- panels inferred visually (layout/grid/white space)
→ DO NOT assign a/b/c
→ save as:
  Fig01_panel_candidate_01.png

IMPORTANT:
- ALWAYS keep the full figure as FigXX.png
- NEVER replace full figure with panels

==================================================
VERIFICATION RULES
==================================================

A figure can be exported as final FigXX ONLY if:

- source evidence exists
- figure index is known
- image corresponds to that figure
- confidence >= medium

Otherwise:
→ move to candidates/
